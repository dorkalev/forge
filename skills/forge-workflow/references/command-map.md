# Forge Command Map (Codex)

Use this map to choose the right Forge command spec quickly.

- Start new ticket work: `references/commands/start.md`
- Load and spec a ticket: `references/commands/load.md`
- Create issue from description: `references/commands/new-issue.md`
- Capture planning into a ticket: `references/commands/capture.md`
- Create worktree for existing branch: `references/commands/worktree.md`
- Main finalize flow: `references/commands/finish.md`
- Generate tests: `references/commands/add-tests.md`
- Fix PR bot findings (CodeRabbit/Greptile/Aikido): `references/commands/fix-pr.md`
- Fix CI compliance failures: `references/commands/fix-compliance.md`
- Compliance check (read-only): `references/commands/audit.md`
- PR compliance body build: `references/commands/verify-pr.md`
- Document hotfix and backport: `references/commands/hotfix.md`
- Release to production: `references/commands/release.md`
- Generate release media (PDF/MP3/MP4): `references/commands/release-media.md`
- Cleanup merged worktree/branch: `references/commands/cleanup.md`
- Bulk cleanup old branches/worktrees: `references/commands/suggest-cleanups.md`
- Update docs table of contents: `references/commands/update-docs-toc.md`
- Update domain documentation: `references/commands/update-domain-docs.md`
- Help routing and command summaries: `references/commands/help.md`

Codex compatibility notes:
- Replace Claude-only command invocations with equivalent shell/MCP/API actions.
- Replace AskUserQuestion with direct user chat questions.
- Keep workflow outputs aligned with Forge expectations.
